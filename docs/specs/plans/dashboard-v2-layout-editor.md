# 대시보드 레이아웃 편집기 (v2 — 미구현 기능)

> **상태**: MVP 이후 구현 예정. 현재 구현 범위 아님.  
> **참조**: `requirements.md §11.5`, `.omc/plans/dashboard-feature.md`

---

## 개요

MVP에서는 위젯 숨기기/표시 + 기본 설정값만 제공한다.  
v2에서는 대시보드 자체에서 WYSIWYG 방식으로 레이아웃을 직접 편집하는 기능을 추가한다.

---

## v2 추가 기능 목록

### 1. WYSIWYG 레이아웃 편집 모드

- 대시보드 헤더 우측 **"레이아웃 편집"** 버튼으로 진입
- 편집 모드 활성 시 버튼 텍스트 → "완료", 헤더 강조 표시
- 완료 / 취소 버튼으로 종료

### 2. 위젯 드래그앤드롭 재배치

- 2열 그리드 내 드래그앤드롭으로 위젯 순서 변경
- 각 위젯 상단에 컨트롤 바 노출: `⠿` 드래그 핸들 / 위젯명 / 눈 아이콘 / 잠금 아이콘
- 위젯 외곽선 점선 표시 (편집 영역 시각화)
- KPI 카드 · 현황 카드는 항상 상단 고정 (드래그 핸들 비활성화)
- 기술 스택: `@dnd-kit/core` 또는 `react-beautiful-dnd` 검토 필요

### 3. 위젯 폭 조절

- 위젯별 1열 / 2열 전환 토글
- 2열 그리드 기준

### 4. 위젯 높이 조절

- 단계별 높이 조절 (단계 수 및 px 값은 구현 시 결정)

### 5. 항목별 잠금 🔒 (Admin 전용)

- Admin이 특정 항목을 잠그면 개인 설정에서 해당 항목 오버라이드 불가
- 잠긴 항목은 편집 모드에서 회색 처리 + 잠금 아이콘 표시
- 잠금 대상: 위젯 순서 / 위젯 폭 / 위젯 높이 / 위젯 숨기기

### 6. dashboard_settings 스키마 확장

MVP 스키마에 아래 컬럼 추가 필요:

```sql
widget_order   JSONB,  -- [{ widget_id, col, row }]
widget_sizes   JSONB,  -- { widget_id: { width: 1|2, height: 1|2|3 } }
locked_fields  JSONB   -- Admin 기본값 행에만 유효: { widget_id: ['order','size','visibility'] }
```

### 7. 위젯 canonical ID 목록

레이아웃 저장/복원에 사용할 위젯 ID (구현 시 확정):

| ID | 위젯명 |
|----|--------|
| `kpi` | KPI 카드 (8종) |
| `distribution` | 분포 탭 |
| `priority_matrix` | 우선순위×상태 매트릭스 |
| `heatmap` | 드릴다운 히트맵 |
| `weekly_trend` | 주간 트렌드 |
| `tag_distribution` | 태그별 분포 |
| `system_cards` | 시스템/메뉴 현황 카드 |
| `assignee_table` | 담당자별 처리 현황 |
| `aging_vocs` | 장기 미처리 Top 10 |

### 8. 설정 충돌 해소 정책

Admin이 기본값을 변경할 때 기존 개인 설정과 충돌 처리 방식 결정 필요:
- **옵션 A**: 잠긴 항목만 강제 반영, 비잠금 항목은 개인 설정 유지
- **옵션 B**: Admin 변경 시 전체 개인 설정 초기화 (강제)
- 추천: 옵션 A (개인 설정 존중)

## 구현 선행 조건

- MVP 커스터마이징(위젯 숨기기/표시)이 안정적으로 운영된 후 진행
- `dashboard_settings` 스키마 마이그레이션 계획 수립 (기존 MVP 데이터 보존)
- 드래그앤드롭 라이브러리 선정 및 번들 크기 검토
