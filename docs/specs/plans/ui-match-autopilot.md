# UI 일치 자동화 — Autopilot 프롬프트

> prototype vs 구현체 UI/UX 99% 이상 일치를 달성하기 위한 반복 실행 프롬프트.
> 이 문서를 업데이트하면 다음 실행 시 자동으로 반영됨.

---

## 실행 명령

```
/oh-my-claudecode:autopilot
```

프롬프트 본문은 아래 "## 프롬프트 본문" 섹션을 그대로 복사.

---

## 프롬프트 본문

```
## 목표
VOC 페이지 구현체(frontend/)를 prototype/과 UI/UX 99% 이상 일치하도록 수정.

---

## Phase 0 — 벤치마크 스크린샷 수집 (최초 1회, 이미 있으면 스킵)

1. Playwright로 prototype dev server 기동
2. 아래 모든 상태를 스크린샷 → .omc/benchmark/ 저장
   - 메인(전체 VOC 리스트), VOC 상세 드로어 열린 상태, VOC 전체화면 모달
   - 대시보드, 각 네비바 항목 클릭 후 상태(하이라이트 포함)
   - 테이블 컬럼 헤더 클릭 후 정렬 토글 상태
   - 대시보드 카드 클릭 후 VOC 필터링된 리스트 상태
   - 기타 팝업·모달·드롭다운 등 인터랙티브 상태 전부
3. 파일명 규칙: bench_<page>_<state>.png

---

## Phase 1 — 구현체 스크린샷 수집

- 동일 상태 목록 기준으로 frontend/ 구현체 스크린샷 → .omc/impl/ 저장
- 파일명 규칙: impl_<page>_<state>.png (벤치마크와 1:1 대응)

---

## Phase 2 — UI/UX 전문가 5인 리뷰 (병렬)

각 전문가가 아래 관점 3가지로 benchmark vs impl 각 쌍을 체크리스트 평가:

| 전문가 | 관점 A | 관점 B | 관점 C |
|--------|--------|--------|--------|
| Expert 1 (레이아웃) | 여백·간격 (8px 그리드 준수) | 컬럼 너비·정렬(좌/우/가운데) | 태그·뱃지 위치 |
| Expert 2 (타이포·색상) | 폰트 패밀리·크기·굵기 | 색상 토큰 사용 여부 | 전체 색조(tone) 일관성 |
| Expert 3 (네비게이션) | 네비바 하이라이트 상태 | 드릴다운·토글 확장 동작 | 선택 상태 시각적 피드백 |
| Expert 4 (테이블/리스트) | 컬럼 헤더 정렬 토글 | 행 클릭 반응 (드로어·전체화면) | 빈 상태·로딩 상태 |
| Expert 5 (기능 완전성) | 누락된 인터랙션 목록 | 대시보드 → VOC 리스트 이동 | 네비바 전환 시 리스트 상태 유지 |

각 항목: 점수(1-5) + 구체적 diff 설명 + 수정 지시 기술
결과 → .omc/reviews/phase2_review.md

---

## Phase 3 — 수정 (병렬 에이전트, 최대 5 이터레이션)

독립적 도메인별로 분리하여 병렬 실행:

| 에이전트 | 담당 |
|---------|------|
| Agent A | 네비바 — 하이라이트, 드릴다운, 확장 애니메이션 |
| Agent B | VOC 리스트 테이블 — 컬럼 너비, 정렬, 태그 위치, 헤더 정렬 토글 |
| Agent C | VOC 상세 — 드로어, 전체화면 모달, 내부 레이아웃 |
| Agent D | 대시보드 — 전면 개편, 카드 클릭 → 필터된 리스트 이동 |
| Agent E | 전역 — 여백·타이포·색상 토큰, design.md 토큰 불일치/누락 수정 |

각 에이전트:
- 수정 후 impl 스크린샷 재촬영 → benchmark와 비교
- 일치율 99% 미만이면 자동 재수정 (최대 5회)
- 완료 시 .omc/reviews/agent_<X>_result.md 기록

---

## Phase 4 — 기능 완전성 체크

prototype 구현 기능 중 impl에서 누락된 것 전수 확인:
- [ ] VOC 행 클릭 → 드로어 오픈
- [ ] 드로어에서 전체화면 전환
- [ ] 컬럼 헤더 클릭 → 오름/내림 정렬 토글
- [ ] 대시보드 카드 클릭 → 해당 VOC 필터 리스트로 이동
- [ ] 네비바 항목 전환 시 VOC 리스트 상태 보존 (불필요한 리셋 없음)
- [ ] 기타 prototype에 있으나 impl에 없는 기능 자동 감지 후 추가

---

## Phase 5 — design.md 토큰 동기화

prototype 시각 결과 기준으로:
- 토큰값이 prototype과 다른 경우 → docs/specs/requires/design.md 수정
- prototype에 있는 패턴인데 토큰이 없는 경우 → 신규 토큰 추가
- 반드시 English only, CSS custom property 형식 유지

---

## 실행 지침

- 메인 컨텍스트 최소화: 모든 실작업은 서브에이전트에 위임
- 병렬 우선: 의존성 없는 에이전트는 동시 실행
- 중단 조건: 99% 이상 일치 달성 또는 5 이터레이션 소진
- 진행 로그: .omc/state/phase8_progress.md 에 단계별 기록
```
