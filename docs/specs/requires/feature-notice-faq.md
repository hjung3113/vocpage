### 10.2 사이드바 네비게이션 구조

패턴 A (하단 고정) 적용 — 현재 사이드바 구조를 유지하고, 구분선 아래 하단에 고정 추가:

```
[사이드바]
 ┌─ 상단 고정 ───────────────────┐
 │  전체 VOC / 내 VOC / 담당 VOC │
 │  대시보드                     │
 ├─ 스크롤 영역 (overflow-y:auto) ┤
 │  ── (구분선) ──               │
 │  시스템 트리 아코디언          │
 │  ── (구분선) ──               │
 │  공지사항 / FAQ               │
 │  ── (구분선) ──               │
 │  Admin (관리자만)             │
 ├───────────────────────────────┤
 │  하단 고정: 유저 프로필        │
 └───────────────────────────────┘
```

**스크롤 동작 규칙**

- 상단 "보기" 섹션(전체/내/담당 VOC, 대시보드)과 하단 유저 프로필은 항상 고정
- 시스템 트리·정보·관리자 영역은 하나의 스크롤 컨테이너로 묶음
- 아코디언 펼침은 항상 **아래 방향**으로만 확장; 컨테이너가 넘치면 세로 스크롤 활성화

**overflow 정책**

- 사이드바 루트 컨테이너: `overflow: hidden` — 내부 스크롤 영역 밖으로 콘텐츠가 새어나오지 않도록 차단.
- 상단 "보기" 섹션(전체 VOC / 내 VOC / 담당 VOC / 대시보드): **고정** — `flex-shrink: 0`; 스크롤 컨테이너에 포함하지 않음. 보기 전환 버튼이 콘텐츠 양에 관계없이 항상 노출되어야 하기 때문.
- 하단 시스템 트리 영역만 `overflow-y: auto` (스크롤 가능) — 시스템·메뉴 아이템이 많아져도 보기 섹션을 밀어내지 않음.

#### 10.2.1 네비게이션 활성 상태 규칙

- **보기 탭 (전체/내/담당 VOC):** 클릭 시 해당 탭만 하이라이트. 시스템·메뉴 클릭 시 보기 탭 하이라이트 해제.
- **시스템 항목:** 클릭 시 해당 시스템 행을 accent 색상으로 하이라이트 (하위 메뉴 선택 시 시스템 하이라이트 해제).
- **메뉴 항목:** 클릭 시 해당 메뉴만 하이라이트.
- 보기 탭과 시스템·메뉴는 상호 배타적 — 동시에 활성 상태 불가.

#### 10.2.2 사이드바 카운트 배지

| 항목        | 배지 내용                          | 표시 조건                                                                                                                                                           |
| ----------- | ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 전체 VOC    | 전체 VOC 수                        | Manager/Admin에게만 표시 (User 역할에게는 탭 및 배지 숨김)                                                                                                          |
| 내 VOC      | 현재 사용자가 등록한 VOC 수        | 1개 이상일 때 표시                                                                                                                                                  |
| 담당 VOC    | 본인 `assignee_id`인 미완료 VOC 수 | User=배지·탭 모두 숨김(담당 개념 없음). Dev/Manager/Admin=본인 assignee_id인 미완료 VOC count; 0이면 배지 숨김. 카운트 갱신: VOC 상태 변경 알림 수신 시 invalidate. |
| 시스템 항목 | 해당 시스템 전체 VOC 수            | 항상 표시                                                                                                                                                           |
| 공지사항    | `!` (빨간 배지)                    | 긴급(urgent) 공지가 오늘 노출 기간 내 존재할 때                                                                                                                     |

### 10.3 공지사항 요구사항

#### 10.3.1 데이터 필드

| 필드            | 타입                       | 설명                                        |
| --------------- | -------------------------- | ------------------------------------------- |
| 제목            | text                       | 필수                                        |
| 내용            | richtext (Toast UI Editor) | 필수                                        |
| 중요도          | enum: 일반/중요/긴급       | 목록에서 시각적 강조                        |
| 팝업 여부       | boolean                    | 로그인 시 팝업 표시 여부                    |
| 노출 기간(차한) | date range (from~to)       | 기간 외에는 자동 비노출 (삭제 아님)         |
| 노출 여부       | boolean                    | Admin/Manager가 수동으로 보이기/숨기기 제어 |

#### 10.3.2 로그인 팝업 동작

- Admin/Manager가 `팝업 여부 = ON`으로 설정한 공지는 로그인 시 팝업 표시.
- 팝업 대상 공지가 **1건이든 2건 이상이든 동일한 2-panel 모달** 구조로 표시:
  - **좌측 패널**: 팝업 대상 공지 목록 (중요도 내림차순, 제목 + 중요도 배지). 1건일 때도 패널 구조 유지.
  - **우측 패널**: 선택된 공지 상세 내용 (기본 선택: 가장 높은 중요도 공지).
  - **모달 footer**: **"오늘 하루 보지 않기" 체크박스 1개** (미열람 공지 전체에 대해 일괄 적용) + **닫기 버튼**.
    - 체크박스는 항목별 개별 옵션이 **아님** — 모달 전체에 단 하나.
    - 체크 후 닫기 시: `localStorage` 키 `notice_dismiss_until_<userId>` 에 `YYYY-MM-DD`(다음날 날짜) 저장.
    - 다음 로그인 시 해당 키의 날짜 값 ≥ 오늘 날짜이면 팝업 미표시; 날짜가 지났으면 다시 노출.
    - 체크 없이 닫기 시: 키 저장 없음 — 다음 로그인 때 다시 팝업 노출.
  - 좌·우 2-panel 구조는 공지 수에 무관하게 항상 유지.

#### 10.3.3 공지사항 목록 (사용자 뷰)

- 노출 기간 내이고 노출 여부 = ON인 공지만 표시
- 중요도에 따른 배지(일반/중요/긴급) 시각 표시
- 제목 클릭 시 상세 내용 표시 (드로어 또는 인라인 펼침)

#### 10.3.4 Admin/Manager 관리 기능

**진입**: 공지사항 페이지 우측 상단 '관리' 버튼 → `?mode=admin` 쿼리 파라미터 토글 (§10.5 참조). Admin 탭 내 별도 서브탭 없음 (D19 확정, 2026-04-26).

- 관리 모드 진입 시 공지 목록 행에 인라인 **노출 여부 토글** (즉시 반영) + 등록/수정/삭제 액션 노출.
- **삭제**: Soft Delete (목록에서 제거, DB에는 유지). Admin은 관리 목록에서 복원 가능 (복원 권한: Admin 전용 — Manager가 삭제한 공지도 Admin만 복원 가능).
- 노출 기간 종료 시 자동 비노출 (삭제 아님, 관리 목록에는 계속 표시).

### 10.4 FAQ 요구사항

#### 10.4.1 데이터 필드

| 필드      | 타입                       | 설명                                            |
| --------- | -------------------------- | ----------------------------------------------- |
| 질문(Q)   | text                       | 필수                                            |
| 답변(A)   | richtext (Toast UI Editor) | 필수, 이미지/표 등 리치텍스트 지원              |
| 카테고리  | faq_categories 테이블 FK   | Admin이 카테고리 동적 관리 (추가/수정/아카이브) |
| 노출 여부 | boolean                    | Admin/Manager가 항목별 노출 제어                |

#### 10.4.2 FAQ 사용자 뷰

- 아코디언 목록: Q를 클릭하면 A가 펼쳐짐
- **키워드 검색**: 질문/답변 텍스트 전문 검색, 매칭 텍스트 하이라이팅
- 카테고리 필터 탭 (전체 / 카테고리별)
- 노출 여부 = ON인 항목만 표시

#### 10.4.3 Admin/Manager 관리 기능

**진입**: FAQ 페이지 우측 상단 '관리' 버튼 → `?mode=admin` 쿼리 파라미터 토글 (§10.5 참조).

- 관리 모드 진입 시 FAQ 항목별 인라인 **노출 여부 토글** (즉시 반영) + 등록/수정/삭제 + 카테고리 관리 액션 노출.
- **삭제**: Soft Delete (목록에서 제거, DB에는 유지). Admin은 관리 목록에서 복원 가능.
- **권한 분기**: Manager는 FAQ 항목 R/W 가능, 카테고리 관리는 읽기 전용(추가·편집·삭제 불가). Admin만 카테고리 CRUD 가능.

#### 10.4.4 FAQ 카테고리 관리 탭 (Admin only)

관리 모드(`?mode=admin`)에서 FAQ 페이지 우상단에 탭 **["FAQ 항목" | "카테고리 관리"]** 표시. "카테고리 관리" 탭은 `role = admin`인 사용자에게만 렌더링.

**카테고리 관리 탭 구성 — Admin Table:**

| 컬럼      | 설명                                        |
| --------- | ------------------------------------------- |
| 이름      | 카테고리 표시명                             |
| 순서      | 목록 노출 순서 (숫자 입력 또는 드래그 정렬) |
| 표시 여부 | ON/OFF 인라인 토글 (즉시 반영)              |
| 액션      | 편집 / 삭제 버튼                            |

- **추가**: 테이블 상단 "카테고리 추가" 버튼 → 모달 (이름·순서·표시 여부 입력).
- **편집**: 행 우측 편집 버튼 → 동일 모달 (기존 값 pre-fill).
- **삭제 제약**: 해당 카테고리에 속한 FAQ 항목이 1건 이상 존재하면 삭제 차단 — 에러 토스트 표시(`"해당 카테고리에 FAQ 항목이 있어 삭제할 수 없습니다."`). 항목이 0건일 때만 삭제 허용.
- BE 엔드포인트: `GET/POST /api/faq-categories`, `PATCH/DELETE /api/faq-categories/:id` (§10.5.1 role guard 적용).

### 10.5 관리 진입점 (D19 확정, 2026-04-26)

> 기존 'Admin 페이지 확장' (공지/FAQ 서브탭 추가) 안은 폐기. 진입 동선을 단축하기 위해 각 페이지 내부에서 관리 모드를 토글하는 방식으로 변경.

#### 10.5.1 진입 위치 및 권한

- 공지사항 페이지·FAQ 페이지 각각의 **우측 상단 페이지 헤더 액션 슬롯**에 '관리' 버튼 노출.
- 가시성: `role ∈ {admin, manager}`인 사용자에게만 렌더링. User/Dev에게는 DOM 자체 미생성 (조건부 렌더 아님 — 컴포넌트 트리에서 제외).
- BE는 관리 API 엔드포인트(`POST/PATCH/DELETE /api/notices`, `/api/faqs`, `/api/faq-categories`)에서 role guard로 이중 방어.

#### 10.5.2 모드 토글 contract

- 관리 모드 진입은 URL 쿼리 파라미터 `?mode=admin`으로 표현 (deep-link / 뒤로가기 / 새로고침에 의해 보존).
- 관리 모드 종료는 동일 페이지의 '읽기 모드' 버튼 또는 쿼리 제거.
- 관리 모드 ON 시 같은 라우트 안에서 인라인으로 관리 액션(등록 폼·인라인 토글·수정/삭제 버튼) 노출. 별도 라우트 분리는 MVP에서 채택하지 않음 (NextGen 재검토 가능).
- 보안: 관리 모드 DOM·핸들러는 `role` 가드 컴포넌트 내부에서만 lazy-load (admin-only 코드가 user 번들로 들어가지 않도록 코드 분할 권장 — 구현 phase 가이드).

#### 10.5.3 관리 기능 (공지/FAQ 공통)

| 액션               | 권한           | 비고                                             |
| ------------------ | -------------- | ------------------------------------------------ |
| 목록 / 등록 / 수정 | Admin, Manager | 공지=중요도/노출기간 입력, FAQ=Q/A/카테고리 입력 |
| 노출 토글          | Admin, Manager | 인라인 즉시 반영                                 |
| Soft Delete        | Admin, Manager | 관리 목록에서는 계속 보임                        |
| 복원               | Admin only     | Manager가 삭제한 항목도 Admin만 복원             |
| FAQ 카테고리 관리  | Admin only     | `faq_categories` CRUD                            |

### 10.6 prototype.html 반영 범위

1. 사이드바 하단에 공지사항/FAQ 메뉴 아이템 추가
2. 공지사항 페이지: 목록(중요도 배지 포함) + 상세 펼침 + **우측 상단 '관리' 버튼 (Admin/Manager 가시)**
3. FAQ 페이지: 아코디언 + 검색 입력창 + **우측 상단 '관리' 버튼 (Admin/Manager 가시)**
4. 로그인 팝업 시뮬레이션: 페이지 로드 시 팝업 모달 표시 + "오늘 하루 보지 않기" 체크박스
5. ~~Admin 페이지: 공지사항/FAQ 관리 탭 추가~~ → **폐기**. Admin 페이지에서 공지/FAQ 서브탭은 제거되며, 관리 동선은 §10.5의 페이지 내 버튼으로 일원화.
