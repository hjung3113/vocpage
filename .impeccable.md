# Impeccable Design Context

## Design Context

### Users

개발자·기획자·PM이 하루 종일 쓰는 B2B 내부 업무 도구. VOC(Voice of Customer) 아이템을 검토·처리하는 팀. 빠른 정보 파악과 작업 처리가 핵심이며, 화려함보다 밀도와 명확성이 중요.

### Brand Personality

precise · systematic · unimpressed (장식 없이 일한다)

### Aesthetic Direction

- **UI**: Linear — 정보 밀도 높고, 군더더기 없고, 섹션 레이블은 작은 대문자+자간, 프로퍼티는 가로 배열(label | value 한 줄)
- **UX**: Jira — 활동 탭은 타임라인 구조, 히스토리는 연결선 있는 피드, 코멘트는 아바타+텍스트
- **Theme**: light-dark 둘 다 지원 (기존 light-dark() 토큰 기반 유지)
- **Font**: Pretendard Variable (기존 --font-ui), D2Coding (이슈 코드)
- **Anti-reference**: 둥근 카드+그림자 남발, 좌측 컬러 스트라이프, 그라디언트 텍스트, 모든 곳에 아이콘

### Design Principles

1. **정보 우선** — 장식보다 내용이 먼저 보여야 한다
2. **수평 프로퍼티** — 라벨과 값은 같은 줄에, 라벨은 고정 너비로 정렬
3. **섹션 레이블은 조용하게** — 작은 대문자, 매우 흐린 색으로 계층 암시
4. **활동은 타임라인** — 연결선으로 시간 흐름 표현
5. **토큰만 사용** — hex/raw OKLCH 금지, var(--\*) 전용
